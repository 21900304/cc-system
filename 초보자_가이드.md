## 📖 이 프로젝트가 뭔가요?

**클로드 코드(Claude Code)**는 AI 비서 "클로드"를 프로그래밍 작업에 도와주는 도구예요.

이 프로젝트는 **클로드를 더 똑똑하게 만드는 "확장팩"**을 모아둔 저장소예요. 마치 게임 캐릭터에 장비를 끼우면 더 강해지듯이, 클로드에 이것들을 추가하면 더 다양한 일을 할 수 있게 돼요!

### 핵심 3가지 요소

1. **스킬(Skills)** 📚 - 클로드가 특정 작업을 잘하도록 가르치는 "교재"
2. **에이전트(Agents)** 🤖 - 특정 일을 전문으로 하는 "AI 조수"
3. **문서(Docs)** 📄 - 사용 설명서와 참고 자료

## 🎯 각 파일이 뭐하는 건가요?

### 1️⃣ Skill (스킬) = 클로드 교육 교재

**위치**: `.claude/skills/폰더이름/`

스킬은 클로드에게 "이런 일은 이렇게 해!"라고 가르치는 교재예요.

#### 예시: `skill-creator` 스킬

```
skill-creator/
├── SKILL.md              ← 핵심 교재 (필수!)
├── scripts/              ← 실행 가능한 프로그램
│   ├── init_skill.py     ← 새 스킬 생성기
│   ├── package_skill.py  ← 스킬 포장기
│   └── quick_validate.py ← 스킬 검사기
└── references/           ← 추가 참고 자료
    ├── workflows.md      ← 작업 흐름 설명
    └── output-patterns.md ← 출력 패턴 모음
```

**SKILL.md가 뭔가요?**
- 클로드가 그 스킬을 언제 사용해야 하는지 알려주는 파일
- 마치 요리책의 "이 요리는 이럴 때 만들어요" 설명 같은 거예요

**scripts 폰더는?**
- Python 프로그램을 모아둔 곳
- 클로드가 직접 실행해서 작업을 자동화해요

---

### 2️⃣ Agent (에이전트) = 전문 AI 조수

**위치**: `.claude/agents/파일이름.md`

에이전트는 특정 일을 전문으로 하는 "AI 직원"이에요.

#### 예시: `brand-logo-finder.md`

```markdown
---
name: brand-logo-finder
description: 브랜드 로고를 찾아주는 전문가
tools: WebFetch, WebSearch
model: haiku
---

당신은 브랜드 로고 찾기 전문가입니다.
사용자가 브랜드 이름을 주면 Brandfetch에서 로고를 찾아주세요.
```

**이게 뭔 뜻인가요?**
- `name`: 이 AI 조수의 이름
- `description`: 언제 이 조수를 부를지 설명
- `tools`: 이 조수가 사용할 수 있는 도구들
- `model`: 이 조수의 "뇌" (haiku=빠름, sonnet=똑똑)

**실제로 어떻게 쓰나요?**
```
사용자: "애플 로고 좀 찾아줘"
클로드: brand-logo-finder 에이전트를 불러옵니다!
에이전트: 애플 로고를 찾았습니다! (링크 제공)
```

---

### 3️⃣ Docs (문서) = 설명서

**위치**: `docs/cc/`

클로드 코드의 기능을 상세히 설명하는 긴 설명서예요.

- `sub-agent.md` → AI 조수 만드는 법 상세 설명
- `hooks.md` → 자동화 규칙 만드는 법 설명
- `slash-commands.md` → 명령어 만드는 법 설명

---

### 4️⃣ Prompt (프롬프트) = 특별한 명령어

**위치**: `prompt/`

클로드에게 내리는 특별한 지침서예요.

#### 예시: `crystalize-prompt.md`
```
# 프롬프트 압축 요청

긴 AI 지침을 짧게 압축해주세요!
```

**왜 필요한가요?**
- AI는 긴 글을 읽을 수 있는 양이 제한적이에요
- 긴 지침을 짧게 줄이면 더 많은 걸 알려줄 수 있어요

---

### 5️⃣ AGENTS.md = AI에게 주는 매뉴얼

**위치**: 프로젝트 루트에 있음

이 파일은 "이 저장소에서 일하는 AI 여러분, 이렇게 작업하세요!"라고 알려주는 규칙서예요.

**뭐가 들어있나요?**
- 파일 이름은 어떻게 짓나요?
- 코드는 어떻게 작성하나요?
- 스크립트는 어떻게 실행하나요?
- 한국어로 소통하세요!

```
│  2단계: 전문 AI 조수 만들기                              │
│                                                         │

│         ↓                                               │
```

**생성된 파일 예시:**
```markdown
---
name: code-reviewer

tools: Read, Grep, Glob, Bash
model: sonnet
---

당신은 시니어 코드 리뷰어입니다.

- 버그는 없나요?
- 보안 문제는 없나요?
- 잘못된 부분은 없나요?
```

**사용 방법:**
```

클로드: code-reviewer 에이전트를 호출합니다!
에이전트: (코드를 분석해서 결과 반환)
```

```json
      }
    ]
```

## 💡 핵심 개념 쉽게 이해하기

### 1. Skill (스킬) = 요리책 📖

**비유:** 요리사(클로드)에게 "이 요리는 이렇게 해!"라고 알려주는 레시피북

**언제 필요한가요?**
- 클로드가 특정 작업을 더 잘하도록 가르치고 싶을 때
- 예: "PDF 파일 다루는 법", "특정 API 사용법"

---

### 2. Agent (에이전트) = 전문 직원 👨‍💼

**비유:** 회사의 전문 부서들
- 코드 리뷰팀 → code-reviewer 에이전트
- 디자인팀 → brand-logo-finder 에이전트
- 테스트팀 → test-runner 에이전트

**언제 필요한가요?**
- 특정 일을 자주 맡기고 싶을 때
- 그 일을 전문으로 하는 "AI 직원"을 만들고 싶을 때

---

### 3. Hook (훅) = 자동화 장치 ⚙️

**비유:** 스마트홈의 자동화 규칙
- "문이 열리면 불을 켜줘"
- "파일을 수정하면 자동으로 저장해줘"

**언제 필요한가요?**
- 특정 상황마다 매번 같은 작업을 반복할 때
- 예: 코드 저장할 때마다 자동으로 포맷팅

---

### 4. Slash Command (슬래시 명령어) = 단축키 ⌨️

**비유:** 핸드폰의 빠른 명령
- "/날씨" → 날씨 알려줘
- "/리뷰" → 코드 리뷰해줘

**언제 필요한가요?**
- 자주 사용하는 명령을 간단하게 치고 싶을 때
- 예: "/review"라고 치면 코드 리뷰 시작

## 🚀 시작하기: 나만의 스킬 만들어보기

### Step 1: 준비하기
```bash
# 1. 터미널(명령어 입력창)을 열어요
# 2. 프로젝트 폴더로 이동해요
cd cc-system
```

### Step 2: 새 스킬 생성하기
```bash
# 새 스킬 만들기 명령어 실행
python3 .claude/skills/skill-creator/scripts/init_skill.py 나의첫스킬 --path .claude/skills
```

**출력 예시:**
```
🚀 Initializing skill: 나의첫스킬
   Location: .claude/skills

✅ Created skill directory: .claude/skills/나의첫스킬
✅ Created SKILL.md
✅ Created scripts/example.py
✅ Created references/api_reference.md
✅ Created assets/example_asset.txt

Next steps:
1. Edit SKILL.md to complete the TODO items
2. Customize or delete the example files
3. Run the validator when ready
```

### Step 3: 스킬 내용 채우기
```bash
# SKILL.md 파일을 열어서 내용을 채워요
# (TODO 표시된 부분을 지우고 실제 내용으로 바꿔요)
```

### Step 4: 검증하기
```bash
# 스킬이 제대로 만들어졌는지 검사
python3 .claude/skills/skill-creator/scripts/quick_validate.py .claude/skills/나의첫스킬
```

### Step 5: 포장하기
```bash
# 배포 가능한 파일로 만들기
python3 .claude/skills/skill-creator/scripts/package_skill.py .claude/skills/나의첫스킬
```

**결과:**
```
나의첫스킬.skill  ← 배포 가능한 파일!
```

## ❓ 자주 묻는 질문 (FAQ)

**Q: 스킬과 에이전트의 차이가 뭐예요?**
> A: 스킬은 "교육 교재"예요. 에이전트는 "전문 직원"이에요. 스킬은 클로드에게 가르쳐주는 것이고, 에이전트는 그 일을 대신해주는 AI예요.

**Q: Python을 몰라도 사용할 수 있나요?**
> A: 네! 스킬의 내용만 채우는 거라면 Markdown(간단한 문서)만 알면 돼요. Python은 필요할 때 클로드가 알려줄 거예요.

**Q: 만든 스킬을 어떻게 공유하나요?**
> A: `.skill` 파일로 만들어서 GitHub에 올리거나, 친구에게 파일로 보내면 돼요. 마치 게임 세이브 파일 공유하듯이!

**Q: 에이전트는 언제 자동으로 동작하나요?**
> A: description에 "Use proactively!"라고 쓰면, 클로드가 상황을 보고 알아서 부를 수 있어요.
